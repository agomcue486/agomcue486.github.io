Bloques de construcción

Ejercicio traducido de aquí.

El diseño de una página web a menudo requiere varias columnas de texto e imágenes. Esto significa crear y diseñar múltiples divs... y hacer que los divs vayan donde los deesemos.

En esta lección, usaremos las propiedades float, margin y padding para crear una pirámide de bloques de construcción de juguetes.

Después, usaremos tres clases para cambiar el color de los bloques.

Resumen

  1. Empezamos con la plantilla de HTML y CSS.
  2. Salimos de una única columna. Instrucciones para explorar float y margin.
  3. Crea la pirámide final de bloques.
  4. Crea y usa tres clases para agregar color a los bloques.

Plantilla HTML y CSS

Utilizamos los archivos HTML y CSS de la plantilla que puedes descargar haciendo clic aquí.

Tenemos algunas propiedades nuevas, como las esquinas redondeadas (propiedad border-radius). Debemos dedicar algún tiempo a examinar la plantilla.

Vemos la página en el navegador

Al abrir la plantilla deberíamos ver una columna de seis bloques de construcción marcados desde la A a la F.

Los seis bloques están hechos con seis divs. Y de forma predeterminada, los divs son elementos de bloque: crean un salto (una nueva línea) para cada uno. Aparecen como una sola columna de elementos.

Salimos de solo una columna

La gracia de los elementos float es que no siempre queremos una sola columna de elementos. Por ejemplo, en este ejercicio, vamos a crear una pirámide de bloques de construcción.

Usamos la propiedad float

Para hacer que los divs estén uno al lado del otro en una fila, usamos la propiedad float.

Podemos decirle a un elemento que flote en la izquierda o en la derecha.

En el archivo CSS, agregamos la línea float: left; a la clase .b que está asociada con el div que contiene la letra B (la sintaxis nueva está en negrita):

                .b {
                    float: left;
                }

Vemos la página

Esto no parece que esté bien. En lugar del bloque B flotando junto al bloque A, el bloque C ahora cubre el bloque B (nota: en algunos navegadores, el bloque B cubre el bloque A).

Aunque parece incorrecto, es la forma en que funciona la propiedad float. La relación de los elementos que están flotando con el resto es extraña: los elementos que no son flotantes, se colocan con independencia de los que sí flotan, apareciendo cosas extrañas.

El bloque C se pinta encima del bloque B. ¿Por qué? Los elementos que no son flotantes no reconocen los elementos que sí flotan y las cosas se vuelven inestables.

Hacemos que sean float: left los seis bloques

En nuestro archivo CSS, añadimos float: left; a los seis divs.

Todos los divs, del .a al .f tienen la misma sintaxis (la sintaxis nueva está en negrita):

                .a {
                    float: left;
                }

Vemos la página

Esto tiene mejor pinta, pero todavía queda, ya que el bloque F no está al lado del bloque E y el fondo blanco ha desaparecido.

Nuevamente, esto es incorrecto, pero es la forma en que funciona la propiedad float. Si los elementos que van contiguos no caben en una línea, pasan a la siguiente.

Y, dado que los elementos sin que no son float no reconocen a los elementos que sí lo son, el div de la clase main_container no reconoce los elementos floats dentro de él. Esto causa un problema porque el div de la clase main_container está actualmente configurado en altura automática (es lo que sucede de forma predeterminada, si no se especifica una altura, un elemento tiene altura automática).

Altura automática significa que main_container solo se abrirá hasta la altura de lo que contenga ... y dado que los bloques están flotando, main_container no los reconoce. No sabe que están ahí.

El div main_container solo se abre a la altura de lo que contenga... y dado que los bloques están flotando, el div main_container no sabe que están allí.

Añadimos overflow: hidden al div main_container

Para que recuperemos el fondo blanco, debemos forzar al div main_container a reconocer los elementos floats que tiene dentro.

Para hacer esto, agregamos la línea overflow: auto; al div .main_container (la sintaxis nueva está en negrita): 

                .main_container {
                    width: 600px;
                    margin-top: 20px;
                    margin-right: automático;
                    margin-left: automático;
                    background-color: .ffffff;
                    overflow: auto;
                }

Esto le dice al main_container que mire más detenidamente lo que hay dentro. Reconocerá los elementos floats y se abrirá para incluirlos.

Vemos la página

Si ahora vemos la página así, todo está funcionando correctamente.

Todos los bloques de construcción ahora son float: left, y el div main_container con el fondo blanco ha regresado.

Experimentamos con la propiedad float

Vamos a hacer ahora float: right los seis bloques

¿Qué sucede si ponemos flotantes los seis bloques en la derecha?

Vamos a cambiar el estilo de los seis divs, del .a al .f con la misma sintaxis nuevamente. Cambiamos el float: left; por float: right; (la sintaxis nueva está en negrita):

            .a{
            float:izquierda derecho ;
            }

Vemos la página

¡Los bloques han cambiado de lugar! El bloque A flota a la derecha, el bloque B flota a la derecha junto al bloque A, y así sucesivamente. Y nuestro amigo el bloque F todavía no cabe en la primera fila, así que pasa a la siguiente.

¡Los bloques han cambiado de lugar!

Usamos los dos float: left y float: right;

¿Qué sucede si ponemos A, B, C flotando a la izquierda y D, E, F a la derecha?

Dale a los divs .a .b y .c la misma sintaxis: float: left.

        .a {
            flotar: izquierda ;
        }

Dale a los divs .d, .e y .f la misma sintaxis: float: right. 


    .d {
        float: right ;
    }

Vemos la página

Los bloques A, B y C están nuevamente en el lado izquierdo del main_container. Mientras que los bloques D, E y F continúan flotando hacia la derecha. El bloque F pobre todavía no cabe en la primera fila, así que pasa a la siguiente. Date cuenta que hay un pequeño hueco que separa los divs que están a la izquierda de los que están a la derecha.

A, B, C tienen el estilo float: left, y D, E, F tienen el estilo float: right.

Creamos varias filas

A veces, una sola fila de elementos no es mejor que una sola columna de elementos.

Usamos clear: left;

Si nuestros elementos están flotando a la izquierda, usamos clear: left para enviarlos a la siguiente fila.

Si nuestros elementos están flotando a la izquierda, la propiedad clear: left;, le dice a un elemento que no puede cruzarse con ningún otro elemento que flote a la izquierda que se haya definido antes en la misma página web. Por tanto, el elemento que tiene la propiedad clear: left se desplaza un poco para abajo ocupando una nueva fila. Como ya hemos visto, los elementos que no son float no reconocen a los que sí lo son. Por tanto, gracias a esta nueva propiedad, un elemento que no esté flotando ya sí los reconoce.

Y sin embargo, en este ejercicio se da la siguiente paradoja. Un elemento puede tener la propiedad clear: left; para no cruzarse con ningún otro elemento float: left anterior. Pero él mismo, puede volver a ser float: left;.

Vamos a crear la estructura que definimos a continuación, aunque no sea el objetivo de este ejercicio, donde Tenemos dos bloques de construcción en cada fila.

  • AB
  • CD
  • EF

Por tanto, para realizar esta última estructura tenemos que:

  1. al primer elemento de una nueva fila ponle el atributo clear: left;.
  2. ese mismo primer elemento de la fila, que también tenga float: left;.
  3. el resto de elementos de la misma fila solo tiene float: left;.

Eso significa que, en esta prueba de dos bloques por fila, necesitamos un clear: left; en los divs .c y .e, que son los primeros elementos de las nuevas filas. No lo ponemos en .a porque es el primer elemento de una nueva fila al ser, precisamente, el primero.

A continuación mostramos la sintaxis de .a, .b y .c. Intenta si puede hacer .d, .e y .f. 

                .a{
                    float: left;
                }

                .b{
                    float: left;
                }

                .c{
                    clear: left;
                    float: left;
                }

Vemos la página

Si hemos hecho el clear:left; y float: left correctamente, nuestra página debería verse así. Si no es así, inténtelo de nuevo.

Movemos los bloques a su posición

En una pirámide, cada fila está centrada en la fila que está debajo.

Utilizamos la propiedad margin para mover los bloques

Todos los elementos tienen margin y padding.

margin es el espacio fuera de un elemento. Es el espacio entre dos elementos.

padding es el espacio dentro de un elemento. Es el espacio entre el borde de un elemento y el contenido dentro de él.

Si nos ayuda a recordar cuál es cuál, pensemos en un sobre acolchado para enviar algo frágil. El padding está en el interior del sobre.

Para mover el bloque A hacia el centro del main_container, agregamos un poco de margin a la izquierda: 

                .a{
                    float: left;
                    margin-left: 150px;
                }

Y para que el bloque se sienta un poco más como un bloque de construcción, agregamos un pequeño margin inferior:

.a{
                float: left;
                    margin-left: 150px;
                    margin-bottom: 2px;
                }

Vemos la página

La propiedad margin-left: 150px agregó espacio fuera de .a, y movió el bloque A más de 150px.

La propiedad margin-lbottom: 2px agregó algo de espacio entre el bloque A y el elemento debajo de él.

Si hemos realizado los márgenes correctamente, nuestra página debería verse así. Si no es así, sigue intentándolo.

Agregamos espacio sobre la pirámide

La pirámide parece apretada en main_container. Podemos agregar espacio encima con margin o padding.

Usamos padding para agregar espacio

Cómo saber cuándo usar el margin y cuándo usar el padding.

La respuesta es insatisfactoria: "depende".

Hemos utilizado margin al mover los bloques de construcción, ya que necesitamos añadir espacio fuera de los bloques. Si usamos padding-left en el bloque A sucedería algo como lo siguiente:

En este caso, la decisión es clara. Cuando usamos margin: left para mover el bloque A fue la mejor decisión. Padding-left no hace lo que se necesitaba.

Pero al agregar espacio sobre el bloque A, la decisión no está clara. Podemos utilizar margin-top en el bloque A. O podemos usar padding-top en main_container. De cualquier manera, obtenemos los mismos resultados.

Como no hemos usado padding en este ejercicio, añadimos este espacio agregando padding-top al div main_container así (la sintaxis nueva está en negrita):

#main_container {
    ancho: 600px;
    margin superior: 20px;
    margin derecho: automático;
    margin: left: automático;
    color de fondo: .ffffff;
    desbordamiento: oculto;
    padding-top:40px;
    }

Añadimos color a los bloques

Para hacer que los bloques sean como juguetes, vamos a añadir color a las letras y a los bordes.

Usamos una clase para añadir el color

Lo mejor de una clase personalizada es que la podemos reutilizar.

Si definimos correctamente una clase personalizada, podemos reutilizar para modificar varios elementos en una página. Por ejemplo, si hacemos una clase llamada "azul" e indicamos la propiedad "border-color: blue;", podemos usar la misma clase tanto en la letra como en el borde.

En el fichero CSS tenemos que poner lo siguiente.

    .azul{
        color:dodgerblue;
        border-color:dodgerblue;
        }

Usamos la clase en el código HTML para hacer que tanto el bloque A como el D sean azules. Y la clase la usamos en dos lugares en cada bloque: para la clase que ya está en el div y para el encabezado h2.

Debemos poner especial atención a la sintaxis. Démonos cuenta que podemos añadir más de una clase a un elemento separando los nombres de las clases que asociamos a un elemento HTML mediante un espacio: (la sintaxis nueva está en negrita) 

    <div id="a" class="border azul">
    <h2 class="azul">a</h2>
    </div>

Vemos la página

Si has aplicado la clase "azul" correctamente nuestra página debería parecerse a la imagen de abajo.

La clase "azul" se ha añadido a las letras A y D y a los bordes que las contienen. ¿Por qué esto ha funcionado? ¿Por qué no añadimosotro borde en las letras? La clase no especifica ni un grosor ni un estilo para el borde; solo qué color usar. Sin grosor ni estilo especificados, un borde no se añade a las letras.
