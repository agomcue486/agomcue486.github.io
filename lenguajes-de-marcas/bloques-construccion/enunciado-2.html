Y, como los elementos sin float no reconocen elementos sin float… el main_container no reconoce los elementos flotados dentro de él. Esto causa un problema porque main_container está actualmente configurado para una altura automática (por defecto, si no especificas una altura, un elemento tiene una altura automática).

Altura automática significa que main_container solo se abrirá hasta la altura de lo que contenga… y como los bloques están flotados, main_container no los reconoce. No sabe que están allí.

Añade overflow:hidden al main_container

Para recuperar el fondo blanco, necesitas forzar que main_container reconozca los elementos flotados dentro de él.

Para hacer esto, añade la línea overflow:hidden; a tu #main_container (nueva sintaxis en negrita):

#main_container{
    width:600px;
    margin-top:20px;
    margin-right:auto;
    margin-left:auto;
    background-color:#ffffff;
       overflow:hidden;
    }

Esto le dice a main_container que mire más cuidadosamente qué hay dentro de él. Reconocerá los elementos flotados y se abrirá nuevamente para incluirlos.

Crea Múltiples Filas

A veces, una sola fila de elementos no es mejor que una sola columna de elementos.

Usa clear:left;

If your elements are floated left, clear:left; tells an element to go to the next row.

Pero una vez que el elemento está en la siguiente fila, necesita flotar nuevamente para que la siguiente fila funcione. ¿Por qué? Como se mostró anteriormente en este ejercicio, los elementos sin float no reconocen elementos con float… y acabas de eliminar la relación float:left; entre dos elementos.

El truco es este:

  1. da al primer elemento de una nueva fila un clear:left;
  2. luego colócale un float:left; encima.

CD
EF

Eso significa que necesitarás un clear:left; en #c y #e (el primer elemento de la nueva fila).

A continuación se encuentra la sintaxis para #a, #b y #c. Mira si puedes hacer #d, #e y #f.

#a{
    float:left;
    }

    #b{
    float:left;
    }

    #c{
    clear:left;
    float:left;
    }

Mueve los Bloques a Su Posición

En una pirámide, cada fila está centrada en la fila de abajo.

Usa Margin para Mover los Bloques

Todos los elementos tienen margin y padding.

Margin es el espacio fuera de un elemento. Es el espacio entre dos elementos. Padding es el espacio dentro de un elemento. Es el espacio entre el borde de un elemento y el contenido dentro de él.

Si te ayuda a recordar cuál es cuál, piensa en un sobre acolchado para enviar algo frágil. El padding está en el interior del sobre.

Para mover el bloque A hacia el centro del main_container, añade algo de margin-left a él:

  #a{
    float:left;
        margin-left: 150px;
    }

Y para hacer que el bloque se parezca más a un bloque de construcción, añade un poco de bottom-margin a él:

  #a{
    float:left;
    margin-left: 150px;
        margin-bottom: 2px;
    }

Ve Tu Página

Margin-left: 150px; añadió espacio fuera de #a y movió el bloque A 150px. Margin-bottom: 2px; añadió espacio entre el bloque A y el elemento de abajo. Si has hecho los margins correctamente, tu página debería verse algo así. Si no, inténtalo de nuevo.

A menudo me preguntan cuándo usar margin y cuándo usar padding.

La respuesta insatisfactoria es, "depende."

Usaste margin cuando movías los bloques de construcción, porque necesitabas añadir espacio fuera de los bloques. Si usaras padding-left en el bloque A, se vería algo así:

En este caso, la decisión es clara. Usar margin-left para mover el bloque A fue la mejor decisión. Padding-left no haría lo que se necesitaba.

Pero cuando añades espacio encima del bloque A, la decisión no es clara. Puedes usar un margin-top en el bloque A. O puedes usar un padding-top en el main_container. De cualquier forma, obtendrías los mismos resultados independientemente de tu decisión.

Añade Color a los Bloques

Para hacer que los bloques se parezcan más a bloques de juguete, vas a añadir color a las letras y bordes. Lo harás usando clases.

Usa una Clase para Añadir Color

Lo mejor de una clase es que puedes usarla una y otra vez.

Y, si la defines correctamente, puedes usarla para modificar diferentes elementos en la página. Por ejemplo, si haces una clase llamada "azul" y le dices que tenga el color azul y el border-color azul, puedes usar la misma clase tanto en la letra como en el borde.

En el CSS, crea la siguiente clase:

  .blue{
    color:dodgerblue;
    border-color:dodgerblue;
    }

Usa la clase en el HTML para hacer que tanto el bloque A como el bloque D sean azules. Añade la clase en dos lugares en cada bloque: a la clase ya existente en el div, y a la h2.

Presta particular atención a la sintaxis. Observa que puedes añadir más de una clase a un elemento separando los nombres de clase con un espacio. La nueva sintaxis está en negrita:

<div>
    <h2 class="blue">a</h2>
    </div>

Consejos Rápidos

Floats…
  • Can be left or right. Use float:left or float:right.
  • Can be cleared with clear:left or clear:right. Use on the first element in a row. Follow it with a float:left for float:right so the element will work with others in the row.
  • Need to be consistent. All elements in a row must float or it won’t work. Floated elements don’t recognize non-floated elements.
  • Will make an auto height container close up. Because the container won’t recognize floated elements inside of it. (Fix by adding overflow:hidden; to the container you want to force open).
  • If you want to learn more about floats, Chris Coyner provides a simple, yet thorough explanation at All About Floats.
Margin and Padding…
  • Is space. Margin is the space outside an element (and thus between two elements). Padding is the space inside an element (and thus between the border and content in the element).
  • Can be used interchangeably at times. Which one you should use depends on what you need the element to do.
Classes…
  • Can be used over and over.
  • Can be used to modify an ID.
  • If defined properly, can be used to modify different kinds of elements.
